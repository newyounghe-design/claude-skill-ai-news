# AI News Skill 设计文档

## 创作背景

作为一个 AI 应用开发者和内容创作者，我每天需要跟踪 AI 领域的最新动态。但信息源太分散：

- Twitter/X 上的 AI 大佬动态
- 各公司官方博客
- GitHub 上的新项目
- YouTube 上的教程视频
- 中文圈的讨论

手动浏览这些信息源既耗时又容易遗漏重要内容。

## 设计思路

### 核心问题

1. **信息过载**：每天有大量 AI 相关内容，但真正重要的只有少数
2. **缺乏深度**：大多数新闻聚合只是罗列，没有分析和判断
3. **工具发现难**：好用的 AI 工具散落在各处，很难及时发现
4. **个性化不足**：通用新闻简报无法针对个人需求筛选

### 解决方案

#### 1. 多源聚合 + 智能筛选

```
数据源                    筛选逻辑
─────────────────────────────────────────
Twitter AI 公司    ──┐
Twitter AI 大佬    ──┼──→  重要性分级  ──→  S/A/B/C
Twitter 中文圈     ──┤     (S 级必报道)
RSS (Anthropic)    ──┤
HuggingFace 论文   ──┤
36kr 快讯          ──┤
YouTube 频道       ──┘
```

**重要性分级标准：**
- **S 级**：新模型发布、重大产品功能、公司战略
- **A 级**：功能更新、技术论文、大佬重要观点
- **B 级**：工具推荐、中文圈讨论
- **C 级**：日常转发（跳过）

#### 2. 深度分析框架

对于 S/A 级新闻，不只是报道，还要回答：

```
┌─────────────────────────────────────────┐
│  新闻事件                                │
├─────────────────────────────────────────┤
│  🔮 影响推演                             │
│  ├─ 短期（1-3个月）：会发生什么          │
│  ├─ 中期（3-12个月）：行业如何演变       │
│  └─ 对你的影响：你应该怎么做             │
├─────────────────────────────────────────┤
│  💭 我的观点                             │
│  └─ 独立判断 + 理由                      │
└─────────────────────────────────────────┘
```

这个框架的价值在于：
- 不只是「发生了什么」，而是「意味着什么」
- 帮助用户快速判断是否需要关注
- 提供可执行的行动建议

#### 3. 工具发现机制

被动等待 RSS 推送不够，需要主动搜索：

```
搜索源              关注内容
────────────────────────────────
GitHub Trending  →  新的 Skill、MCP Server、Agent 框架
YouTube          →  AI 工具教程、Claude Code 技巧
X/Twitter        →  带 github.com 链接的推文
```

**个性化推荐逻辑：**

```python
def should_recommend(tool, user_profile):
    # 高度相关
    if tool.type in ['Claude Skill', 'MCP Server']:
        return '✅', '直接提升 Claude Code 效率'
    if tool.category == '视频创作':
        return '✅', '辅助电影解说工作'
    if tool.category == 'Obsidian':
        return '✅', '增强知识管理'

    # 不推荐
    if tool.price > 50 and not tool.has_clear_advantage:
        return '❌', '性价比不高'
    if tool.stability == 'early_stage':
        return '❌', '过于早期'

    # 一般相关
    return '⚠️', '可选，非核心需求'
```

#### 4. 上下文管理策略

**问题**：完整简报内容太长，会污染对话上下文，影响后续交互质量。

**解决方案**：分离存储和展示

```
执行流程
─────────────────────────────────────────
Step 1-3: 静默执行（不输出到对话）
    ↓
Step 4: 生成两份内容
    ├─→ 完整简报 → 保存到 Obsidian 文件
    └─→ 快讯摘要 → 展示在对话中
```

**对话中只展示：**
```markdown
# 🤖 AI 快讯 - 03月14日

1. **标题** - 一句话 [链接]
2. ...
5. ...

💡 今日一句：...
📄 完整简报已保存：news/2026-03-14_智能简报.md
```

这样做的好处：
- 对话上下文保持干净
- 详细内容永久保存在 Obsidian
- 用户可以随时查阅完整版本

## 技术实现

### 数据抓取

使用 RSSHub 作为统一的 RSS 代理：

```bash
# 本地运行 RSSHub
docker run -d --name rsshub -p 1200:1200 diygod/rsshub

# 抓取 Twitter 用户
curl "http://localhost:1200/twitter/user/AnthropicAI"

# 抓取 YouTube 频道
curl "http://localhost:1200/youtube/user/@Fireship"
```

**为什么用 RSSHub：**
- 统一接口，简化代码
- 支持大量数据源
- 本地运行，无 API 限制

### 数据清洗

RSS 数据通常包含 HTML 标签和转义字符：

```python
# 清洗规则
clean_rules = {
    '<br>': '\n',
    '<img ...>': '',
    '&lt;': '<',
    '&gt;': '>',
    '&amp;': '&',
    '&#39;': "'",
}

# 过滤规则
skip_if = [
    lambda item: item.title == '',           # 空标题
    lambda item: item.is_pure_retweet,       # 纯转发
    lambda item: is_emoji_only(item.text),   # 纯表情
]
```

### 输出格式

使用 Markdown 格式，兼容 Obsidian：

```markdown
---
date: 2026-03-14
tags: [ai-news, daily-briefing]
created: 2026-03-14 15:30
---

# 🤖 AI 简报 - 03月14日

## 📌 今日重点
...
```

**Frontmatter 的作用：**
- `date`：方便按日期检索
- `tags`：支持 Obsidian 标签系统
- `created`：记录生成时间

## 迭代历程

### v1.0 - 基础版本
- 只有 RSS 抓取
- 简单罗列新闻
- 全部输出到对话

**问题**：信息太多，没有重点

### v2.0 - 增加筛选
- 引入重要性分级
- 只报道 S/A 级新闻

**问题**：只有事实，没有分析

### v3.0 - 增加分析
- 添加影响推演
- 添加观点输出
- 添加工具推荐

**问题**：输出太长，污染上下文

### v4.0 - 分离存储
- 完整简报存文件
- 对话只展示快讯
- 静默执行中间步骤

### v4.1 - 工具发现
- 增强工具发现（主动搜索）
- 个性化推荐说明
- 优化输出格式

### v4.2 - 当前版本
- 增加命令系统（类似 /handoff）
- 优化配置表格式
- 完善文档和示例

## 未来计划

1. **历史追踪**：记录每条新闻的后续发展
2. **趋势分析**：基于历史数据识别长期趋势
3. **自动执行**：每天定时运行，推送到手机
4. **多用户支持**：支持不同用户背景的个性化配置

## 设计原则总结

1. **信息密度优先**：宁可少报，不可乱报
2. **观点要有依据**：每个判断都要说明理由
3. **个性化是核心**：通用推荐没有价值
4. **上下文是资源**：不要浪费在中间数据上
5. **文件是持久化**：重要内容必须落盘
